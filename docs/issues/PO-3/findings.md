# Findings - [Content] DOM 셀렉터 자동 복구(PO-3)

## 📅 2026-02-01

### 문제 정의
트레이딩 사이트는 자주 UI가 업데이트되며, 하드코딩된 단일 셀렉터는 유지보수에 취약함.

### 해결 전략: Multi-layer Selector
각 요소(버튼, 가격 등)에 대해 여러 가지 매칭 전략을 준비함:
1.  **Direct ID/Class**: 가장 빠르고 표준적인 방식.
2.  **Aria Label/Text**: 텍스트 기반 매칭 (UI가 바뀌어도 텍스트는 덜 바뀜).
3.  **Hierarchy Path**: 특정 부모 아래의 자식 관계.

### Auto-healing Flow
1.  기본 셀렉터로 시도 -> 실패.
2.  백업 리스트에서 유효한 요소를 찾음.
3.  찾은 요소의 새로운 셀렉터를 추출하여 로컬 저장소에 캐싱.
4.  다음번 실행 시 캐싱된 성공 셀렉터를 최우선 사용.
